AWDev Plugins Updater - Changelog
==================================

0.1.2 - 2026-03-18
-------------------
- Removed: error logging calls
- Removed: dead add-plugin JS code (no UI counterpart existed)

0.1.1 - 2026-03-17
-------------------
- Fixed: fix_folder_name(): crash when ZIP extracts without a subfolder (flat structure)
  rename() was called with target path inside the source directory -> Invalid argument
  Source/remote-source paths are now normalised and compared to detect flat ZIPs
  Rename is redirected to a sibling directory instead
- Fixed: plugins_loaded hook priority raised to 20 for reliable init order
- Changed: rename_source() now receives fully resolved target path directly
- Changed: awdev_get_local_version() caches get_plugins() via wp_cache_get/set (group awdev_updater)
  get_plugins() now scans filesystem at most once per request

0.1.0 - 2026-03-17
-------------------
- Fixed: awdev_fetch_api_data(): null/empty API body now explicitly treated as failure and cached as false
- Fixed: saveToggle() in settings.js: missing .then()/.catch() meant failures were silently ignored
- Fixed: Re-check button: version cell resets to ... before fetch and shows ? on error
- Fixed: compareVersions(): Number() replaced with parseInt() to safely ignore pre-release suffixes
- Added: register_activation_hook() now calls awdev_activate() for first-activation defaults
- Added: awdev_sync_auto_update_defaults(): adds missing awdev_auto_updates entries for built-in plugins
- Added: escHtml() in settings.js now also escapes apostrophes
- Changed: awdev_activate() delegates to awdev_sync_auto_update_defaults()
- Changed: AWDev_Updater::get_remote_data() delegates to shared awdev_fetch_api_data() helper
- Changed: Removed unused jQuery dependency from wp_enqueue_script()
- Changed: JS Unicode escapes replaced with literal UTF-8 characters in PHP files
- Also included (from 0.0.9): json_last_error() validation, error_log() on rename failure,
  days display in last-checked, author field from API, uninstall.php, awdev_built_in_plugins(),
  generate-l10n.yml workflow, update_option() refactor, uninstall.php consolidation

0.0.8
-----
- Fixed: Auto-update filter returned null instead of true for AWDev plugins
- Fixed: Plugin folder rename silently failed when target directory already existed

0.0.7
-----
- Changed: Improved DarkAdmin compatibility on the settings page
- Changed: Removed colour-related !important declarations from base selectors
- Changed: Kept DarkAdmin-specific override rules for reliable dark mode rendering

0.0.6
-----
- Fixed: Language file msgid strings now match the corresponding __() calls in settings.php
- Fixed: Corrected two mismatched strings
- Fixed: Updated Project-Id-Version in all .po and .pot files to 0.0.6

0.0.5
-----
- Fixed: Plugin folder rename failed on bulk updates and WP auto-updates
- Added: Fallback matching by extracted source folder name
- Changed: Extracted rename logic into private rename_source() method

0.0.4
-----
- Changed: Replaced wp_redirect() with wp_safe_redirect()
- Changed: Added wp_unslash() and absint() for POST input sanitisation
- Changed: Added translators comments for _n() and printf() i18n calls
- Changed: Replaced direct rename() with WP_Filesystem::move()
- Changed: Added phpcs:ignore for intentional direct DB queries

0.0.3

- Added: Global auto-update toggle mirrors state to all per-plugin toggles

- Changed: Save Settings button moved into Auto-Update Settings card

0.0.2

- Fixed: Dark Mode styling compatibility with DarkAdmin
